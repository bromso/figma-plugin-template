<task type="auto">
  <name>Task 1: Install Biome and create root biome.json</name>
  <files>package.json, biome.json</files>
  <read_first>
    - package.json (to see current devDependencies)
    - 04-RESEARCH.md Pattern 1 (biome.json config structure)
  </read_first>
  <action>
1. Install Biome at the repo root with exact version pinning:
   ```bash
   bun add -D -E @biomejs/biome
   ```
   This installs a single binary — no plugins needed. The `-E` flag is critical to avoid version mismatch warnings (per RESEARCH.md Pitfall 1).

2. Create `biome.json` at the repo root with the following content:
   ```json
   {
     "$schema": "https://biomejs.dev/schemas/2.4.10/schema.json",
     "linter": {
       "enabled": true,
       "rules": {
         "recommended": true
       }
     },
     "formatter": {
       "enabled": true,
       "indentStyle": "space",
       "indentWidth": 2,
       "lineWidth": 100
     },
     "javascript": {
       "formatter": {
         "quoteStyle": "double",
         "semicolons": "always",
         "trailingCommas": "es5"
       },
       "jsxRuntime": "transparent"
     },
     "files": {
       "includes": ["**/*.ts", "**/*.tsx", "**/*.js", "**/*.jsx", "**/*.json"],
       "ignore": ["**/node_modules", "**/dist", "**/.turbo", "**/bun.lock"]
     }
   }
   ```

   Key config choices:
   - `indentStyle: "space"` + `indentWidth: 2` matches existing codebase convention
   - `jsxRuntime: "transparent"` is correct for React 17+ (no `import React` needed) — satisfies LINT-02
   - `files.ignore` uses `**/` prefix for each pattern (Biome 2.x no longer auto-prepends `**/`) — per RESEARCH.md Pitfall 2
   - SCSS files are NOT in `files.includes` — Biome does not support SCSS (documented gap in STATE.md)
   - `bun.lock` is ignored because it is a binary file

3. Verify the schema URL resolves. If it 404s during biome check, fall back to removing the `$schema` field (per RESEARCH.md Assumption A4). Run: `bunx biome check --max-diagnostics=10 .` to test. Expect either zero diagnostics or only auto-fixable style issues on existing code.

4. If Biome reports lint/format errors on existing code, run `bunx biome check --write .` to auto-fix. Review the changes to ensure they are purely style (indentation, quotes, semicolons, trailing commas). Do NOT auto-fix if changes affect logic or semantics.
  </action>

    <automated>cd /Users/jonasbroms/Sites/figma-plugin-template && test -f biome.json && bunx biome check --max-diagnostics=5 . 2>&1 | head -20</automated>
  </verify>
