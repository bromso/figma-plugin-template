    **Step 1 — Determine Vitest binary path:**

    Run from workspace root:
    ```bash
    ls node_modules/.bin/vitest packages/common/node_modules/.bin/vitest packages/ui/node_modules/.bin/vitest 2>/dev/null
    ```

    If vitest is at `node_modules/.bin/vitest` (root hoisted by Bun), use `${workspaceFolder}/node_modules/.bin/vitest` as the program path. If only at per-package paths, use `${workspaceFolder}/node_modules/vitest/vitest.mjs` as fallback.

    **Step 2 — Create .vscode/launch.json with three configurations:**

    ```json
    {
      "version": "0.2.0",
      "configurations": [
        {
          "type": "node",
          "request": "launch",
          "name": "Debug Vitest (current file)",
          "autoAttachChildProcesses": true,
          "skipFiles": ["<node_internals>/**", "**/node_modules/**"],
          "program": "${workspaceFolder}/node_modules/.bin/vitest",
          "args": ["run", "${relativeFile}"],
          "smartStep": true,
          "console": "integratedTerminal"
        },
        {
          "type": "chrome",
          "request": "launch",
          "name": "Debug UI (Chrome)",
          "url": "http://localhost:5173",
          "webRoot": "${workspaceFolder}/packages/ui/src",
          "skipFiles": ["<node_internals>/**"]
        },
        {
          "type": "chrome",
          "request": "attach",
          "name": "Attach to Figma Plugin (Developer VM)",
          "port": 9222,
          "urlFilter": "*figma*",
          "skipFiles": ["<node_internals>/**"],
          "pathMapping": {
            "/": "${workspaceFolder}/apps/figma-plugin/src/"
          }
        }
      ]
    }
    ```

    **Configuration details:**

    1. **Debug Vitest (current file)** — Open any `.test.ts` or `.test.tsx` file and press F5. Vitest runs that single file with Node debugger attached. `autoAttachChildProcesses: true` is needed because Vitest forks worker processes. `smartStep: true` skips non-user code automatically.

    2. **Debug UI (Chrome)** — First run `bun run dev:ui-only` in terminal, then launch this config. Opens Chrome at the Vite dev server URL. Set breakpoints in React components.

    3. **Attach to Figma Plugin (Developer VM)** — Advanced. Requires launching Figma with `--remote-debugging-port=9222`. Attaches VS Code to Figma's plugin sandbox JS engine. Most developers will use Figma's built-in "Open Console" instead.

    **Step 3 — Update .gitignore to whitelist launch.json:**

    Read .gitignore. Check if `.vscode/launch.json` is already allowed. The Phase 4 summary noted that `.vscode/settings.json` and `.vscode/tasks.json` were whitelisted. If launch.json is not yet whitelisted (i.e., .vscode/ is ignored but only settings.json and tasks.json are negated), add `!.vscode/launch.json` on the line after the other `.vscode/` whitelist entries.

    Look for a pattern like:
    ```
    !.vscode/extensions.json
    !.vscode/settings.json
    !.vscode/tasks.json
    ```
    Add `!.vscode/launch.json` in the same block.

    If .vscode/ is not ignored at all, no gitignore change is needed.

    **Step 4 — Verify the file is valid JSON:**

    ```bash
    node -e "JSON.parse(require('fs').readFileSync('.vscode/launch.json','utf8')); console.log('Valid JSON')"
    ```
