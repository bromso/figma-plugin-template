# Phase 1 Plan 1: Monorepo Root Scaffolding Summary

**One-liner:** Turborepo + Bun workspace root with `tasks` pipeline, replacing flat npm/pnpm project with `bun@1.3.11` as sole package manager.

## What Was Built

Replaced the existing flat single-package Vite project root with a Turborepo + Bun monorepo workspace configuration. The root `package.json` now declares workspace globs for `apps/*` and `packages/*`, delegates all scripts to `turbo run`, and sets `packageManager` to `bun@1.3.11`. A `turbo.json` defines 5 task types with correct caching and topological dependency ordering. Empty `apps/` and `packages/` directories are tracked by git via `.gitkeep` files. Old lockfiles (`package-lock.json`, `pnpm-lock.yaml`) were deleted and replaced with `bun.lock` generated by `bun install`.

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| `packageManager: "bun@1.3.11"` (no v prefix) | Turborepo validates against `(bun\|npm\|pnpm\|yarn)@\d+\.\d+\.\d+`; v prefix causes hard failure |
| `tasks` key in turbo.json (not `pipeline`) | `pipeline` was deprecated in Turborepo 2.0 and silently removed in 3.0 |
| `bun.lock` committed to repo | Enables deterministic installs; Turborepo reads it for dependency-change-based cache invalidation |
| `version` field removed from root package.json | Private workspace roots are never published to npm; version field is vestigial |
| All old devDependencies removed from root | Existing deps (react, vite, etc.) move to `apps/figma-plugin` in Phase 2 |
| `"private": true` required | Bun refuses workspace processing without this flag |
