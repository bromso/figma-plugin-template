- `apps/figma-plugin/` — The Figma plugin application

**# figma-plugin-template**

One-line description: Figma/FigJam plugin boilerplate with React, Vite, TypeScript, and Turborepo monorepo.

**## Features**
- Turborepo monorepo with Bun package manager
- React + TypeScript + Vite build pipeline
- Single-file HTML output (required by Figma plugin API)
- 14 native Figma UI components via react-figma-ui
- Storybook for component documentation
- Vitest for testing (happy-dom for UI, node for common)
- Biome for linting and formatting
- Type-safe message passing between plugin and UI (monorepo-networker)

**## Quick Start**
```bash
# Clone the template
git clone https://github.com/jonasbroms/figma-plugin-template.git
cd figma-plugin-template

# Install dependencies
bun install

# Start development
bun run dev
```

**## Project Structure**
```
apps/
  figma-plugin/     -- Figma plugin app (Vite builds to dist/)
  storybook/        -- Storybook component documentation
packages/
  common/           -- Shared types and network events (@repo/common)
  ui/               -- UI components, styles, and app shell (@repo/ui)
```

Explain that packages are JIT source-only (export raw TypeScript). Use `@repo/*` workspace imports.

**## Commands**
Table format showing all available commands:
| Command | Description |
|---------|-------------|
| `bun run dev` | Watch mode (plugin + UI via Turborepo) |
| `bun run build` | Production build |
| `bun run test` | Run all tests |
| `bun run test:watch` | Watch mode for tests |
| `bun run lint` | Lint with Biome |
| `bun run storybook` | Start Storybook |
| `turbo run build-storybook` | Build static Storybook (cached) |

**## Architecture**
Explain the two-process model:
- Plugin side (apps/figma-plugin/src/plugin/) — Figma sandbox, API access, no DOM
- UI side (packages/ui/src/) — iframe, React app, react-figma-ui components
- Common (packages/common/src/) — shared types, network event definitions

Describe messaging with monorepo-networker: define events in networkSides.ts, build channels in *.network.ts, use emit/request pattern.

**## UI Components**
List all 14 react-figma-ui components available via `@repo/ui`: Button, Checkbox, Disclosure, DisclosureItem, Icon, IconButton, Input, Label, SectionTitle, OnboardingTip, Radio, Select, Switch, Textarea, Type.

Mention Storybook: `bun run storybook` to browse components interactively.

**## Plugin Configuration**
- Edit `apps/figma-plugin/figma.manifest.ts` to set your plugin name, ID, and permissions
- Get your plugin ID from Figma (Plugins > Manage plugins > Create new plugin)

**## Testing in Figma**
1. Run `bun run build`
2. Open Figma desktop
3. Plugins > Development > Import plugin from manifest
4. Select `apps/figma-plugin/dist/manifest.json`

**## Tech Stack**
Table: Bun 1.3.11, Turborepo, Vite 6, React, TypeScript, Biome 2.4.10, Vitest 4.x, Storybook 8.6.18
