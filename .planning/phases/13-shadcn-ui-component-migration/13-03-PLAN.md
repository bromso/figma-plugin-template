2. Rewrite `packages/ui/src/app.tsx` to use the new shadcn/ui component APIs. Key API changes:
   - `Button tint="primary"` becomes `Button variant="default"` (default is primary-styled)
   - `Button` (no tint) becomes `Button variant="secondary"`
   - `Checkbox` needs a separate `Label` sibling: `<div className="flex items-center gap-2"><Checkbox id="cb1" defaultChecked /><Label htmlFor="cb1">Option A</Label></div>`
   - `Radio` becomes `RadioGroup` + `RadioGroupItem` + `Label`:
     ```tsx
     <RadioGroup defaultValue="a">
       <div className="flex items-center gap-2">
         <RadioGroupItem value="a" id="r1" />
         <Label htmlFor="r1">Option A</Label>
       </div>
       <div className="flex items-center gap-2">
         <RadioGroupItem value="b" id="r2" />
         <Label htmlFor="r2">Option B</Label>
       </div>
     </RadioGroup>
     ```
   - `Switch` needs adjacent `Label`: `<div className="flex items-center gap-2"><Switch id="sw1" defaultChecked /><Label htmlFor="sw1">Enable feature</Label></div>`
   - `Select` becomes composable:
     ```tsx
     <Select defaultValue="a">
       <SelectTrigger><SelectValue placeholder="Choose..." /></SelectTrigger>
       <SelectContent>
         <SelectItem value="a">Option A</SelectItem>
         <SelectItem value="b">Option B</SelectItem>
       </SelectContent>
     </Select>
     ```
   - `Disclosure` + `DisclosureItem` becomes `Accordion` + `AccordionItem` + `AccordionTrigger` + `AccordionContent`:
     ```tsx
     <Accordion type="single" collapsible defaultValue="item-1">
       <AccordionItem value="item-1">
         <AccordionTrigger>More info</AccordionTrigger>
         <AccordionContent>Details here.</AccordionContent>
       </AccordionItem>
     </Accordion>
     ```
   - `OnboardingTip` becomes `Alert` + `AlertDescription`:
     ```tsx
     <Alert>
       <Icon iconName="info" className="size-4" />
       <AlertDescription>This is a tip for new users.</AlertDescription>
     </Alert>
     ```
   - `Icon`, `IconButton`, `SectionTitle`, `Type` keep same API

   The app must still render 4 sections: Inputs, Buttons, Display, Layout — with the same text content so existing tests pass.
