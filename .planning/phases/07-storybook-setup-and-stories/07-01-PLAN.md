**apps/storybook/.storybook/main.ts:**
```typescript
import path from 'node:path';
import { mergeConfig } from 'vite';
import type { StorybookConfig } from '@storybook/react-vite';

const config: StorybookConfig = {
  framework: '@storybook/react-vite',
  stories: ['../src/stories/**/*.stories.@(ts|tsx)'],
  addons: ['@storybook/addon-essentials'],
  async viteFinal(config) {
    return mergeConfig(config, {
      resolve: {
        alias: [
          {
            find: /^figma-plugin-ds$/,
            replacement: path.resolve(
              __dirname,
              '../../../packages/ui/__mocks__/figma-plugin-ds.js',
            ),
          },
        ],
      },
    });
  },
};

export default config;
```

IMPORTANT: The addons array must contain ONLY `@storybook/addon-essentials`. Do NOT add `@storybook/addon-docs` — it is bundled inside addon-essentials in Storybook 8.x and listing both causes a duplicate Docs tab or startup warning.

The `viteFinal` alias is required because react-figma-ui internally imports from `figma-plugin-ds` which has broken ESM bare imports. This reuses the same mock stub that Vitest uses.

Do NOT include `vite-plugin-singlefile` — Storybook runs its own independent Vite instance.

**apps/storybook/.storybook/preview.ts:**
```typescript
import type { Preview } from '@storybook/react-vite';
import '@repo/ui'; // side-effect: loads figma-plugin-ds CSS

const preview: Preview = {

  parameters: {
    viewport: {
      viewports: {
        figmaSmall: {
          name: 'Figma Plugin Small (300x200)',
          styles: { width: '300px', height: '200px' },
        },
        figmaMedium: {
          name: 'Figma Plugin Medium (320x500)',
          styles: { width: '320px', height: '500px' },
        },
        figmaLarge: {
          name: 'Figma Plugin Large (400x600)',
          styles: { width: '400px', height: '600px' },
        },
      },
    },
  },
};

export default preview;
```

IMPORTANT: Import `Preview` type from `@storybook/react-vite` (NOT from `@storybook/react`). The framework package `@storybook/react-vite` exports this type and is the only Storybook React package in devDependencies. Importing from `@storybook/react` would fail because that package is not installed.

Note: In Storybook 8.x the viewport config uses `viewports` (not `options` which is the 10.x API).

Then install dependencies:

```bash
cd apps/storybook && bun install
```

Add two new tasks to `turbo.json`:

1. `"storybook"` task — persistent dev server, no cache (matches the existing `dev` task pattern):
```json
"storybook": {
  "cache": false,
  "persistent": true
}
```

2. `"build-storybook"` task — cacheable static build:
```json
"build-storybook": {
  "dependsOn": ["^build"],
  "outputs": ["storybook-static/**"]
}
```

These go in the `tasks` object alongside the existing `build`, `dev`, `lint`, `test`, and `test:watch` tasks.

After updating turbo.json, verify Storybook starts correctly:
```bash
cd apps/storybook && npx storybook --smoke-test 2>&1 || bun run build-storybook
```

The `--smoke-test` flag (if available in 8.x) starts Storybook and exits 0 if it initializes without errors. If `--smoke-test` is not available, use `bun run build-storybook` as the smoke test — a successful static build proves the config is valid.

Also add a root-level convenience script to `package.json` if desired, but this is NOT required since `turbo run storybook` works from root already.
