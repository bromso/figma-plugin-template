Create 15 story files in `apps/storybook/src/stories/`. All use CSF3 format with `satisfies Meta` pattern. Autodocs is enabled globally in preview.ts so no per-file `tags` needed.

IMPORTANT: Before writing stories, read the react-figma-ui type definitions to get accurate prop names. The library re-exports components with different names than the CONTEXT.md expects (e.g., `SelectMenu` not `Select`, `Onboarding` not `OnboardingTip`, `DisclosureTip` not `DisclosureItem`). Stories import from `@repo/ui` which handles the aliasing.

In Storybook 8.x, `Meta` and `StoryObj` types are imported from `@storybook/react` (NOT `@storybook/react-vite`).

**Pattern for simple components** (Button, Checkbox, Input, Label, SectionTitle, Radio, Switch, Textarea, Type, Icon, IconButton):

```typescript
import type { Meta, StoryObj } from '@storybook/react';
import { Button } from '@repo/ui';

const meta = {
  component: Button,
  title: 'Components/Button',
} satisfies Meta<typeof Button>;

export default meta;
type Story = StoryObj<typeof meta>;

export const Default: Story = {
  args: {
    children: 'Click me',
  },
};
```

Read the actual TypeScript props for each component to populate meaningful `args`. For components that accept `children`, provide a sensible default string. For boolean props (like `isDestructive` on Button), create variant stories.
