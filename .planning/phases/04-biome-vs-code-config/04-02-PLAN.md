  <action>
1. Create `.vscode/` directory at the repo root.

2. Create `.vscode/settings.json` with Biome as default formatter and formatOnSave:
   ```json
   {
     "editor.defaultFormatter": "biomejs.biome",
     "editor.formatOnSave": true,
     "editor.codeActionsOnSave": {
       "source.fixAll.biome": "explicit",
       "source.organizeImports.biome": "explicit"
     },
     "[typescript]": {
       "editor.defaultFormatter": "biomejs.biome"
     },
     "[typescriptreact]": {
       "editor.defaultFormatter": "biomejs.biome"
     },
     "[javascript]": {
       "editor.defaultFormatter": "biomejs.biome"
     },
     "[javascriptreact]": {
       "editor.defaultFormatter": "biomejs.biome"
     },
     "[json]": {
       "editor.defaultFormatter": "biomejs.biome"
     },
     "[jsonc]": {
       "editor.defaultFormatter": "biomejs.biome"
     }
   }
   ```
   Per-language overrides are required because VS Code may have other formatters installed globally. This ensures Biome wins for all TS/JS/JSON file types (per RESEARCH.md Pitfall 5).

3. Create `.vscode/extensions.json` with recommended extensions:
   ```json
   {
     "recommendations": [
       "biomejs.biome",
       "vitest.explorer",
       "antfu.vite"
     ]
   }
   ```
   Vitest and Vite extensions are included now (even though Phase 5 adds Vitest) because VSDX-02 explicitly requires recommending them.

4. Create `.vscode/tasks.json` with dev and build shortcuts:
   ```json
   {
     "version": "2.0.0",
     "tasks": [
       {
         "label": "dev",
         "type": "shell",
         "command": "bun",
         "args": ["run", "dev"],
         "group": "build",
         "isBackground": true,
         "problemMatcher": [],
         "presentation": {
           "reveal": "always",
           "panel": "new"
         }
       },
       {
         "label": "build",
         "type": "shell",
         "command": "bun",
         "args": ["run", "build"],
         "group": {
           "kind": "build",
           "isDefault": true
         },
         "presentation": {
           "reveal": "always",
           "panel": "shared"
         }
       }
     ]
   }
   ```
   `isBackground: true` on dev prevents VS Code from waiting for exit. `problemMatcher: []` is required for background tasks to prevent VS Code from hanging.

  <action>
1. Create `figma-plugin-template.code-workspace` at the repo root:
   ```json
   {
     "folders": [
       { "name": "root", "path": "." },
       { "name": "apps/figma-plugin", "path": "apps/figma-plugin" },
       { "name": "packages/common", "path": "packages/common" },
       { "name": "packages/ui", "path": "packages/ui" }
     ],
     "settings": {
       "editor.defaultFormatter": "biomejs.biome",
       "editor.formatOnSave": true
     }
   }
   ```

   Key design choices:
   - The "root" folder is included so developers can access root-level config files (turbo.json, biome.json, package.json) from the Explorer sidebar.
   - Folder names use the path convention (e.g., "apps/figma-plugin") for clarity in the sidebar.
   - Minimal settings in .code-workspace (just formatter + formatOnSave) — the full per-language overrides live in .vscode/settings.json which VS Code layers underneath.
   - Paths are relative to the .code-workspace file location (repo root).
