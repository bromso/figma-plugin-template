# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: react-figma-ui Integration.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none (react-figma-ui owns the system) | CONTEXT.md — direct re-export strategy |
| Preset | not applicable | No shadcn; not a Next.js/Vite web app |
| Component library | react-figma-ui v1.1.0 (figma-plugin-ds) | CONTEXT.md — 14 named components |
| Icon library | react-figma-ui Icon component (figma-plugin-ds icons) | Inferred from library |
| Font | Inter (loaded by figma-plugin-ds CSS) | figma-plugin-ds default |

> Note: This is a Figma plugin panel, not a browser web app. The design system is Figma's own native UI kit (figma-plugin-ds). No custom design token layer is added. All visual decisions derive from the library's built-in CSS and Figma's plugin UI guidelines.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between sibling elements |
| sm | 8px | Padding inside component groups, gap between label and control |
| md | 16px | Section padding, gap between category groups in sampler |
| lg | 24px | Gap between major sections of the component sampler |
| xl | 32px | Top/bottom padding for the scrollable panel container |

Exceptions:
- Touch/click targets follow react-figma-ui's internal sizing — do not override.
- Plugin panel minimum width is 300px per Figma constraints; the sampler layout must not overflow at 300px.

---

## Typography

All sizes and weights are dictated by figma-plugin-ds CSS. No custom type scale is introduced.

| Role | Size | Weight | Line Height | Source |
|------|------|--------|-------------|--------|
| Body / control label | 11px | 400 (regular) | 1.5 | figma-plugin-ds default |
| Section title | 11px | 600 (semibold) | 1.4 | SectionTitle component |
| Type component (small) | 11px | 400 | 1.5 | Type component, size="small" |
| Type component (large/bold) | 12px | 600 | 1.4 | Type component, size="large", weight="bold" |

> Do not introduce new font sizes. Use the `Type` component (from react-figma-ui) for all text rendering in the demo app rather than bare HTML elements.

## Color

All color values are provided by figma-plugin-ds CSS. Do not hardcode hex values in component code.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF | Plugin panel background, input surfaces |
| Secondary (30%) | #F5F5F5 | Section dividers, grouped row backgrounds |
| Accent (10%) | #18A0FB | Primary Button fill, focused input outline, Switch checked state |
| Destructive | not used in this phase | No destructive actions in static sampler |

Accent reserved for: Primary Button variant only, Switch in checked state, focused Input ring. Do NOT apply accent color to labels, icons, headings, or section titles.

> figma-plugin-ds enforces these colors via its own CSS classes. The demo app uses component props (e.g. `isActive`, `type="primary"`) to activate accent states — no inline color styles.

## Layout Contract

- Outer container: scrollable `<div>` with `overflow-y: auto`, `height: 100%`, padding `md` (16px) sides, padding `sm` (8px) top/bottom.
- Each category group is headed by a `SectionTitle` component.
- Components within a group are stacked vertically with `sm` (8px) gap.
- Gap between category groups: `lg` (24px).
- No horizontal overflow at 300px panel width. All inputs use `width: 100%`.
- No wired-up state: all stateful props (checked, isActive, selected) are hardcoded static values in JSX. No `useState` for demo controls.

## CSS Integration Contract

- `figma-plugin-ds/dist/figma-plugin-ds.css` is imported as a side-effect in `packages/ui/src/index.ts` — not in main.tsx, not in app.tsx.
- Existing `packages/ui/src/styles/main.scss` and all partials under `styles/` are removed (they belong to the old custom demo).
- App-level layout styles (the scrollable container, category group gaps) are written in a new `app.module.scss` using CSS Modules — scoped to app.tsx only.
- BEM classes from figma-plugin-ds use `.figma-` prefix — no collision with CSS Module class names.
- `vite-plugin-singlefile` inlines the CSS at build time; no `<link>` tags in dist output.
